# Location Lock System

## Purpose

This permanent production manual keeps established Village Stories places
spatially, materially, seasonally, and emotionally continuous.

Use it for still-image generation, Google Flow, boards, scene review, and edit
handoff.

It is not permission to invent a location, alter a building, move a landmark, or
modernise early-1990s rural Tamil Nadu.

## Core Rule

Generate the known place, not a generic village that resembles it.

Every location is a repeatable set of geography, zones, materials, landmarks,
access routes, light behaviour, object homes, and human use.

## Series Illustration Standard

Every location must be drawn in the same Village Stories anime style as every
character.

Apply to all visible place elements:

- buildings and walls;
- roofs, doors, thresholds, and courtyards;
- wells, paths, fences, and fields;
- trees, crops, and garden plants;
- stone, mud, wood, plaster, and metal surfaces;
- sky, rain, mist, and distant background.

Use:

- thin clean consistent line art;
- soft cel-style painted shading;
- warm cinematic or weather-appropriate light;
- earthy grounded palette;
- handcrafted animation-background quality.

A location must never look photoreal, stock-photo, 3D-rendered, or visually
detached from the anime characters in front of it.

Match `11_Assets/Character_References/Father_Canon_Design_Reference.png` and
`01_Canon/01_Series_Style_Lock/Art_Style.md`.

## Authority

1. Approved current production decision.
2. `01_Canon/` timeline, continuity, art style, camera, and world direction.
3. Relevant `02_World/Locations/` documents.
4. Relevant `04_Objects/` documents for fixed local objects.
5. Approved location reference, map, board, or selected previous asset.
6. Episode script, storyboard, and scene-state ledger.
7. This manual, then prompt modules and tool settings.

Generated images, tool defaults, and visual convenience have no canon authority.

If a map direction, entrance, permanent object, or material is unresolved,
record `HOLD FOR CANON OR RESEARCH`.

## Permanent Village Orientation

- Family house, rear yard, and kitchen are west.
- Paddy fields and pond are east.
- Village entrance, banyan tree, and river ghat are north.
- Fields, pond, and irrigation canal are south.
- Morning light comes from the east.
- Evening light comes from the west.
- Never reverse sunlight across adjacent shots.
- Never use geography merely as decorative background.
- Preserve readable movement between connected locations.
- Preserve screen direction across an established route.

## Lock Vocabulary

- `PLACE HARD LOCK`: named established location.
- `LAYOUT HARD LOCK`: zone relationship, entrance, exit, and axis.
- `LANDMARK HARD LOCK`: repeated visual anchor.
- `MATERIAL HARD LOCK`: structural material and surface language.
- `STATE FACT`: current wetness, occupancy, light, or seasonal condition.
- `VARIABLE`: documented temporary change.
- `OBJECT HOME`: usual approved placement of an object.
- `CARRYOVER`: incoming visual state.
- `NEGATIVE`: prohibited addition or transformation.
- `OPEN`: unknown fact requiring an owner.

## Location Source Packet

Build this packet for every visible named place.

```text
Episode / scene / shot ID:
Location and exact zone:
Why this location is needed:
Owning canon files and revisions:
Approved reference assets and roles:
Orientation and camera-facing direction:
Arrival route / entry point:
Exit route / next destination:
Permanent layout locks:
Permanent landmarks:
Materials and construction:
Object homes visible in this shot:
Season / time / weather:
Light source and direction:
Surface state:
People, animals, and permitted background activity:
Current temporary changes:
Incoming shot / carryover:
Outgoing shot / required end state:
Camera geography:
Required absences and negatives:

```

Load only facts necessary for the intended frame.

Never ask a model to reconstruct an entire house from one sentence.

## Reference Hierarchy

Use the most specific approved reference first.

1. Written location canon.
2. Approved dedicated location design reference.
3. Approved map, plan, or orientation diagram.
4. Selected preceding shot for continuity.
5. Approved storyboard or location keyframe.
6. Research reference for material fact only.
7. Generated output only as a selected continuity source.

Label each uploaded reference:

```text
LOCATION_LAYOUT
LOCATION_IDENTITY
ZONE_DETAIL
MATERIAL_DETAIL
INCOMING_CONTINUITY
LIGHTING_CONTEXT
OBJECT_HOME
DO_NOT_COPY
```

Do not use a beautiful candidate to replace an approved plan.

Do not infer an unseen room or yard from a tight close-up.

## Immutable Location Facts

Lock these whenever documented and visible:

- Location identity.
- Relative orientation.
- Entry and exit points.
- Fixed rooms and outdoor zones.
- Roof, wall, floor, path, and drainage construction.
- Permanent trees, shrine, bridge, or platform.
- Permanent public utilities.
- Default object homes.
- Repeated camera-facing axes.
- Period-correct infrastructure.
- The location’s ordinary social purpose.

## Variables

Variables require a scene-state entry.

- Time of day.
- Light strength and direction.
- Season.
- Weather.
- Ground wetness.
- Dust.
- Puddle presence.
- Smoke, steam, or cooking fire.
- Temporary work setup.
- Harvest, produce, or laundry state.
- Character occupancy.
- Animal position.
- Open door, shutter, or gate state.
- Festival dressing when source-backed.
- Temporary moved object.

Every variable needs a start state and an end state.

## Family House Lock

The Family House is a lived-in rural home, not a generic cottage or mansion.

Preserve its documented relationship to Front Thinnai, Courtyard, Backyard,
Kitchen, Bedroom, Puja Corner, gardens, and Livestock Shelter.

Treat each zone as distinct even when one wide frame shows several zones.

Keep construction handmade, practical, maintained, and period-correct.

Do not turn it into a polished resort, urban bungalow, ruin, or poverty display.

Record which doorway, threshold, or yard edge a subject uses.

## Front Thinnai Lock

The Front Thinnai is a front-facing resting and receiving space.

Use it for seated waiting, familiar conversation, small domestic pauses, and
practical thundu or footwear moments when source-backed.

Preserve its relation to the house front and village-facing approach.

Do not treat it as an indoor living room, shop counter, or concrete veranda.

Keep its shade and sitting logic readable.

## Courtyard Lock

The Courtyard is an active domestic transition zone.

Use it for approved household work, family circulation, drying, preparation, or
quiet observation.

Preserve usable walking paths between adjacent home zones.

Do not block every route with decorative props.

Do not make it a manicured lawn or empty studio floor.

## Backyard Lock

The Backyard belongs to the west-side home cluster.

Preserve its practical relation to kitchen work, garden, storage, livestock, and
household movement as defined by canon.

Use natural ground, lived-in surfaces, and motivated work traces.

Do not relocate it east of the house for a convenient sunset.

## Front Garden Lock

The Front Garden is an established domestic green edge.

Keep plants, path relationship, and house frontage readable.

Use only seasonally plausible growth and approved gardening activity.

Do not turn it into a formal ornamental estate garden.

Do not add exotic landscaping, plastic pots, or modern lawn furniture.

## Vegetable Garden Lock

The Vegetable Garden supports ordinary food-growing and care.

Show planted rows, soil, water, tools, and produce only when the scene requires
them and source canon supports them.

Keep it useful, modest, and near its documented home relation.

Do not make it a large commercial farm or a decorative flower garden.

## Livestock Shelter Lock

The Livestock Shelter is practical animal-care space.

Keep its materials, animal use, hay, water, and entry logic source-backed.

Show only approved animals and calm ordinary husbandry.

Never manufacture distress, filth spectacle, threat, or overcrowding.

Do not turn it into a modern metal shed.

## Kitchen Lock

The Kitchen is a working room, not a stylised restaurant set.

Load `Kitchen.md` and its architecture, storage, workflow, lighting, camera, and
continuity documents as applicable.

Preserve the raised clay stove, traditional tools, storage logic, smoke and
ventilation behaviour, work surfaces, and the west-side orientation.

Fire, steam, utensils, and ingredients must follow an approved task.

Do not add gas hobs, mixers, refrigerators, glossy tiles, or modern cabinets.

## Kitchen Zone Rules

- Identify cooking, preparation, storage, washing, and serving zones.
- Record which zone contains the principal action.
- Keep the raised clay stove in its canonical relationship to work surfaces.
- Place only required utensils at reachable distances.
- Keep smoke light, motivated, and ventilated.
- Keep flames controlled and ordinary.
- Carry pots and ingredients through plausible routes.
- Preserve light from the documented opening and time.
- Do not move every tool between cuts.
- Do not overload counters with decorative food.

## Puja Corner Lock

The Puja Corner is a respectful established household devotional space.

Use documented religious items and lamp behaviour only.

Keep it modest, clean, and integrated into home life.

Do not add unsupported idols, commercial decoration, flashing electric lights, or
festival-scale spectacle to an ordinary day.

## Bedroom Lock

The Bedroom is a private domestic room.

Use only source-backed furnishings, sleeping arrangements, light, and activity.

Keep it simple, practical, and period-correct.

Do not add modern furniture, electronics, hotel styling, or unsupported personal
objects.

## Village Utilities Lock

Village Utilities are shared stable infrastructure.

Maintain:

- Packed-earth Main Village Path.
- Narrow earthen footpaths.
- Open hand-built stone drainage channels.
- Granite-slab stone culverts.
- Granite benches and stone platforms.
- Wooden village-entrance sign with hand-painted Tamil name.
- Traditional oil lamp posts.
- Wooden community notice board with small tiled roof.
- Compost pits and traditional organic waste practice.
- Bullock-cart resting area.
- Informal bicycle leaning places.

Never add electric bulbs to oil lamp posts.

Never add official traffic signs, concrete paving, or municipal bins.

## Village Square Lock

The Village Square is a public gathering location.

Preserve its relation to notice board, public seating, paths, and ordinary village
movement as source-backed.

Use sparse motivated activity, not a permanent crowd.

Keep bicycles leaning naturally rather than in modern racks.

Avoid market-city density, stage lighting, or unapproved vendors.

## Banyan Tree And Vinayagar Shrine Lock

The Banyan Tree and shrine are permanent northern anchors.

Keep the tree’s scale, shade, roots, seating relationship, and shrine respect.

Keep the shrine modest and culturally grounded.

Use oil-lamp lighting only as canon supports.

Do not remove the tree, replace it with a generic tree, or turn the shrine into a
large temple complex.

## River Ghat Lock

The River Ghat is a north-side water-edge location.

Preserve approach, steps or platform logic, river edge, seating, and relation to
the Banyan-tree area as documented.

Use safe, ordinary, non-spectacular water activity.

Do not create floods, rescues, dangerous currents, or dramatic cliff geography.

## Stone Bridge Lock

The Stone Bridge is a fixed crossing landmark.

Preserve its stone construction, route role, scale, and approach direction.

Use it for believable pedestrian, bicycle, livestock, or cart passage when
source-backed.

Do not replace it with concrete, steel, painted railings, or an oversized bridge.

## Tea Stall Lock

The Tea Stall is an established village social-work location.

Keep its construction, serving area, seating logic, vessel use, and owner
relationship grounded in canon.

Do not turn it into a modern café, branded shop, neon sign, or crowded city stall.

## Small Grocery Store Lock

The Small Grocery Store has a stable village retail identity.

Load its source packet before showing stock, counter, owner, or customer flow.

Use only approved period-correct storage, packaging, and display.

Do not introduce barcodes, refrigerators, digital payment signs, contemporary
brands, or supermarket shelving.

## Primary School Lock

The Primary School is an established community learning place.

Load its architecture, grounds, school-day state, and staff packets.

Keep child activity safe, ordinary, and source-backed.

Do not add modern buses, computers, plastic playground equipment, or urban campus
features.

## Craft Location Locks

Blacksmith Forge, Carpenter Workshop, Potter Clay Yard, and Barber Under the Big
Tree each require their own source packet.

Do not merge their tools, work surfaces, owners, or material language.

Show one readable craft action at a time.

Do not use sparks, sawdust, clay, blades, or tools as action spectacle.

Keep visitor paths, work safety, and object placement plausible.

## Festival Ground Lock

The Festival Ground is used only when the approved event requires it.

Festival dressing, crowd level, lamps, flowers, clothing, and temporary objects
must be sourced to that event.

Remove temporary decoration in a later scene only when the timeline supports it.

Do not make every evening a festival.

## Surface-State Ledger

Track visible surfaces per shot.

```text
Shot ID | Zone | Ground | Wall | Roof | Path | Water | Smoke | Door/gate |
Temporary setup | Start state | End state | Owner
```

Examples of valid state:

- Dry packed earth with light walking dust.
- Damp firm path with edge puddles after monsoon rain.
- Moss on stone after sustained wet weather.
- Light cooking smoke rising from an active clay stove.
- Open wooden shutter during a documented daytime task.
- Closed gate after a person exits.

Examples of invalid state:

- Dry dust immediately after visible heavy rain.
- Blue-black storm mood without story support.
- Pristine polished floor in a working kitchen.
- New concrete path in an established earthen route.
- Electric lamp in an oil-lamp post.

## Prompt Construction Order

1. Name the location and exact zone.
2. State permanent layout and orientation facts.
3. State materials and landmark anchors.
4. State time, weather, light, and surface condition.
5. State required people, object homes, and one action.
6. State camera direction and entry/exit logic.
7. State carryover facts.
8. State negatives.

Use observable terms.

Do not replace layout facts with mood adjectives.

## Location Prompt Template

```text
At [ESTABLISHED LOCATION], in the [EXACT ZONE], preserve [LAYOUT / LANDMARK /
MATERIAL HARD LOCKS]. It is [TIME / SEASON / WEATHER], so [VISIBLE SURFACE AND
LIGHT CONSEQUENCE]. [SUBJECT] performs [ONE ACTION] beside [APPROVED OBJECT
HOME]. Camera faces [DIRECTION] from [PLAUSIBLE POSITION], preserving [ENTRY /
EXIT / SCREEN-DIRECTION CARRYOVER]. Keep [REQUIRED ABSENCES]. Do not show
[LOCATION NEGATIVES].
```

## Kitchen Prompt Example

```text
Inside the established west-side family kitchen, preserve the raised clay stove,
practical preparation and storage zones, traditional tools, and natural working
surfaces. Diffused morning light enters from the east-facing opening and mixes
with a small controlled cooking flame. Mother stirs one approved cooking vessel
at the stove while Father remains at the threshold. Keep the counter reachable,
the tool homes stable, and the route to the courtyard clear. Do not add gas
appliances, refrigerators, glossy cabinets, tiles, or modern packaging.
```

## Village Path Prompt Example

```text
On the established packed-earth Main Village Path, preserve the worn tracks,
grass edges, open stone drainage channel, and shade from native roadside trees.
It is a dry early morning, with east light falling from screen left according to
the approved camera direction. Father walks north toward the Village Square with
the Dog following at a calm distance. Keep the route wide enough for a bullock
cart and do not add asphalt, road markings, street lights, electric poles,
plastic bins, or modern vehicles.
```

## Google Flow Rules

Flow must preserve the approved source location.

Use an approved still for continuity-critical movement.

Use a second image only when the end-state location change needs protection.

State which environmental elements may move.

Permitted movement may include:

- Mild leaf movement.
- Gentle smoke rise.
- Natural rain continuation.
- Small water ripples.
- Soft fabric response.
- One person’s grounded steps.
- One door or shutter action when ledgered.

Keep fixed landmarks fixed.

Do not allow rooms to expand, walls to change, paths to bend, or doors to appear.

Example:

```text
Keep the kitchen architecture, raised clay stove, tool placement, west-side
orientation, and doorway unchanged. Animate only light cooking smoke rising and
Mother completing one slow stir. The camera remains stable. No new doorway,
appliance, utensil, person, or lighting source appears.
```

## Location QA: Identity

- Correct named location.
- Correct exact zone.
- Correct orientation.
- Correct entry and exit relationship.
- Correct permanent landmark.
- Correct material language.
- Correct period infrastructure.
- Correct object homes.
- Correct route scale.
- Correct social purpose.

## Location QA: State

- Time of day matches scene.
- Light direction matches orientation.
- Weather matches adjacent shots.
- Ground wetness is plausible.
- Vegetation matches season.
- Smoke, steam, and fire are motivated.
- Door and gate state match ledger.
- Temporary setup is documented.
- People use plausible circulation routes.
- Object movement is logged.

## Location QA: Camera

- Camera is at plausible human position.
- No drone or fisheye distortion.
- Camera faces documented direction.
- Landmarks anchor geography.
- Screen direction survives the cut.
- Subject has a clear route.
- Foreground does not hide essential location information.
- No impossible room geometry appears.

## Rejection Categories

```text
LOCATION_ID_DRIFT
LAYOUT_DRIFT
ORIENTATION_DRIFT
LANDMARK_DRIFT
MATERIAL_DRIFT
OBJECT_HOME_DRIFT
WEATHER_DRIFT
LIGHT_DRIFT
PERIOD_ERROR
ROUTE_ERROR
FLOW_GEOMETRY_DRIFT
UNSUPPORTED_ADDITION
```

Reject immediately for an altered building layout, moved permanent landmark,
reversed sunlight, wrong route, modern infrastructure, or a new unsupported
location element.

## Revision Protocol

Keep the rejected asset, prompt, references, and QA note.

Name the earliest broken location fact.

Repair the packet before revising prose.

Repair reference role before adding detail.

Change one responsible clause.

Regenerate only the affected shot where possible.

Compare the revision against the incoming and outgoing selected assets.

If the generator cannot preserve a wide location, split into an establishing
still and tighter action shot.

Never accept geometry drift because the mood is attractive.


The editor must not reorder location shots in a way that reverses route or time
logic without a new approved transition.

## Change Control

This manual governs reproduction of existing place canon.

It cannot add a room, road, utility, landmark, shop, shrine, water feature, or
modern service.

Make an approved world-canon decision first.

Then update location canon, map references, object homes, source packets, and
prompt modules together.

## Final Operator Check

- Which known place is visible?
- Which zone is visible?
- What proves its layout?
- Which direction does the camera face?
- Where does the subject enter and exit?
- What has changed since the prior shot?
- What must not change?
- Which modern element must be excluded?

If one answer is absent, hold the shot.

## Guiding Principle

Every return to a location should feel like a return to a real, remembered place
with stable geography and a changing daily life.
