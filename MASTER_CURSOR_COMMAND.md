# Story and World Continuity

Never redesign:

* Characters.
* Locations.
* Architecture.
* Objects.
* Village geography.
* Established relationships.
* Camera language.
* Visual style.
* Prompt architecture.
* Animation workflow.

Story conflict must remain gentle, proportionate, and safe. No villains, horror, cruelty, violence, manipulation, or spectacle of hardship.

Gentle married-life reminiscence is allowed when it supports family history and present care. Romance-focused plots and courtship drama are not part of the series promise.

# Google Flow and AI Rules

For every AI-generated production asset, preserve:

* Recurring character appearance and clothing state.
* Recurring location layout and geography.
* Object identity, materials, and starting condition.
* Camera direction and shot purpose.
* Lighting, weather, season, and time-of-day continuity.
* One-image versus two-image workflow decisions.
* Prompt modules, style tags, and series locks.
* Animation movement and transition continuity.

Google Flow is a production tool. It is never a source of new canon.

Reject a visually attractive result if it breaks an established lock.

# Working Method

Create one file at a time.

For that file:

1. Read the existing documents that own related canon.
2. Define one clear responsibility and scope boundary.
3. Write the full production manual.
4. Check that it does not duplicate another authority.
5. Check all character, location, object, and seasonal references.
6. Validate Google Flow and YouTube implications where relevant.
7. Run available lint and whitespace checks after substantive edits.
8. Review the document critically.

Only then move to the next file.

---

# Self-Review Questions

Before declaring a document complete, ask:

* Would a new production team member know what to do next?
* Would this reduce a real future production mistake?
* Does it preserve established continuity?
* Does it explain why its rules exist?
* Does it improve AI and Google Flow consistency?
* Does it add a unique authority rather than duplicate another file?
* Would a professional animation studio retain this as an internal reference?

If any answer is no, improve the document before moving on.
