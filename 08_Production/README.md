# Production

## Purpose

This folder permanently defines how a Village Stories episode moves from an approved idea to a published and archived YouTube release.

It is the studio handbook for writers, directors, storyboard artists, AI image and animation operators, sound editors, reviewers, and future assistants.

World, Character, Object, Story Engine, and Episode System documentation define what may be made.

This folder defines how to make it consistently.

---

# Production Philosophy

Village Stories production must feel deliberate, patient, and handcrafted even when AI tools are used.

The workflow protects:

* Character identity.
* Location and object continuity.
* Early-1990s rural Tamil Nadu authenticity.
* Calm emotional storytelling.
* Clear, respectful YouTube presentation.
* Long-term asset reuse and revision safety.

Speed is valuable only when it does not weaken the work.

---

# Production Pipeline

```text
Approved episode premise
↓
Pre-production and research
↓
Script and storyboard
↓
Image prompts and reference locks
↓
Google Flow animation
↓
Voice, music, natural sound, and edit
↓
Quality control
↓
Thumbnail, metadata, and publishing
↓
Archive, review, and future reuse
```

Every stage has an input, an owner, a review gate, and a documented handoff. No stage should silently correct a problem created earlier.

# Relationship With Other Documents

Production work inherits authority from:

```text
01_Canon/
02_World/
03_Characters/
04_Objects/
05_Research/
06_Story_Engine/
07_Episode_System/
```

`08_Production/` does not redefine canon. It operationalises canon as repeatable studio work.

The future `09_AI_Prompt_System/` will contain reusable prompt modules. Production will use those modules but remains responsible for validating the finished results.

---

# Google Flow Rule

Google Flow is a production tool, not a source of canon.

Every generated image or animation must begin from approved reference locks, preserve its incoming scene state, and be reviewed before becoming a production asset.

---

# Canon Authority

This folder permanently defines the Village Stories production workflow.

When it conflicts with a higher canon document, the higher document wins. When it conflicts with an approved episode-specific production decision, the documented approval record determines the correct version.

---

# Guiding Principle

Every release should make the Village Stories universe stronger, more trustworthy, and easier to continue.
